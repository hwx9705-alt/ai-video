# Remotion 组件自适应布局改造方案

## 问题描述

渲染出的视频存在两个系统性问题：

1. **内容稀疏时大量空白** — 如 CompareTwo（2:32）只有 3 条 bullet，卡片却拉满整个屏幕高度，下半部全是空白
2. **字体不随内容密度调整** — 不管内容多少，字体都是同一个大小，内容少时没有利用多余空间

## 根本原因分析

| 层面 | 问题 | 涉及组件 |
|------|------|---------|
| **布局拉伸** | 容器用 `flex: 1` + `alignItems: "stretch"`，强制子元素撑满父容器全部高度，不管内容多少 | CompareTwo, FlowSteps |
| **字号静态** | 所有 fontSize 要么来自 theme（固定值），要么直接硬编码（如 `34`、`58`），完全不考虑内容量 | 全部 9 个组件 |
| **无密度感知** | 组件拿到 props（如 `points: string[]`）后，不会根据 `points.length` 来调整字号或间距 | 全部组件 |

### 为什么会「每次都要调」？

因为 LLM 生成的分镜内容量是不可控的 — 有时一个 CompareTwo 段落有 5 条对比点，有时只有 2 条。组件用固定布局渲染，不适配内容量，就必然出现：
- 内容少 → 大量空白 + 字体显小
- 内容多 → 可能溢出或拥挤

要「从系统上解决」，就是让组件**自己感知内容量并调整布局**。

---

## 解决方案：组件级内容自适应

### 核心思路

不改数据契约（LLM 无法可靠生成布局参数），而是在每个组件内部，根据 `props` 中的内容量（条目数、文本长度）**自动计算**字号和间距。

### Step 1: 新建共享工具 `src/responsive.ts`

提供两个缩放函数，供各组件调用：

```typescript
/**
 * 根据条目数计算缩放系数
 *   条目 ≤2 → 1.35x（放大）
 *   条目 3  → 1.2x
 *   条目 4~5 → 1.0x（基准）
 *   条目 >5 → 逐渐缩小（最低 0.85x）
 */
function contentScale(itemCount: number, ideal?: number): number

/**
 * 根据文本长度计算字号缩放
 *   短文本（≤10字）→ 1.3x
 *   中等 → 1.0x
 *   长文本 → 按比例缩小（最低 0.75x）
 */
function textScale(text: string, maxChars?: number): number
```

组件使用方式：
```typescript
// 以 CompareTwo 为例
const scale = contentScale(left.points.length);
const bodySize = Math.round(theme.fontSize.body * scale);  // 3条时 32×1.2 = 38px
const gap = Math.round(22 * scale);                         // 间距同步放大
```

### Step 2: 修复 CompareTwo（最严重）

**当前问题**：
- 三列容器 `flex: 1` → 卡片撑满整屏高度
- 卡片标签 `fontSize: 34`（硬编码，偏小）

**改动**：
1. 移除容器 `flex: 1`，改为 `flexShrink: 0`，让内容决定高度
2. 标签 `34` → `theme.fontSize.subtitle`（42）
3. 引入 `contentScale(points.length)` 缩放 body 字号和 gap
4. 条目少时加大卡片 padding，视觉更饱满

### Step 3: 修复 FlowSteps（中等）

**当前问题**：
- 步骤 label `fontSize: 26`、description `fontSize: 20`（太小）
- `maxWidth: 200` 限制步骤宽度（步骤少时浪费空间）
- 每个步骤 `flex: 1` 平铺整屏

**改动**：
1. label `26` → `theme.fontSize.body`（32）
2. description `20` → `theme.fontSize.label`（26）
3. 移除 `maxWidth: 200`，改为根据步骤数动态计算
4. 步骤数字 badge 联动放大

### Step 4: 修复 BulletList（中等）

**当前问题**：条目 ≤3 时，`fontSize.body`（32）配大量空白，显得内容稀疏

**改动**：
- 引入 `contentScale(items.length)` 缩放字号
- 条目 ≤3 时 body → ~38-40px，gap 加大，bullet dot 同步放大

### Step 5: 统一硬编码字号

多个组件直接写死了 px 值，应改为引用 theme + 缩放：

| 组件 | 当前硬编码 | 改为 |
|------|-----------|------|
| KeyPoint.tsx | 主文字 `58px` | `theme.fontSize.title` + `textScale()` |
| ImageWithOverlay.tsx | 标题 `64px` / 副标题 `34px` | `theme.fontSize.title` / `theme.fontSize.subtitle` |
| CompareTwo.tsx | 标签 `34px` / VS `42px` | `theme.fontSize.subtitle` |
| DataReveal.tsx | 前缀 `56px` / 后缀 `52px` | 基于 `theme.fontSize.hero` 派生 |

### Step 6: 验证

1. 重新渲染完整视频
2. 抽帧检查：CompareTwo（frame 4290）、FlowSteps、BulletList
3. 确认：内容少时字体更大、无大面积空白；内容多时不溢出
