## 一、项目背景与目标

### 1.1 项目定位

仿「小Lin说」风格的 B 站财经/科技科普视频自动化生产系统。
目标：**把一个选题方向变成可发布的视频脚本+分镜+视觉素材**，减少重复性创作劳动，保留人工审核和创意介入点。

### 1.2 核心工作流

```
用户输入选题方向
      ↓
[Research Agent]  知识框架整理
      ↓ 审核门1：选题确认
[Topic Agent]     选题策划 + 角度设计
      ↓ 审核门2a：大纲确认
[Script Agent]    脚本大纲 → 完整脚本 → AI自审循环
      ↓ 审核门2b：完整脚本 + 口播预演
[Storyboard Agent] 分镜方案 + 视觉风格种子
      ↓ 审核门3：分镜方案确认
[Visual Producer]  程序化图表 + AI生图
      ↓ （规划中）
[Audio Agent]     语音合成 + BGM混音
      ↓ （规划中）
[Compose Agent]   ffmpeg 视频合成
      ↓ 审核门4：成片确认
     发布
```

### 2.2 LLM 接入（agents/__init__.py · BaseAgent）

**改造前**：使用 `openai.OpenAI` 客户端（SSL 损坏无法用）

**改造后**：直接 `requests.post` 到 OpenAI 兼容接口

```python
_session = requests.Session()
_session.trust_env = False

def call_llm(self, system_prompt, user_prompt, temperature=None, max_tokens=None):
    payload = {
        "model": self.config.model,
        "messages": [
            {"role": "system", "content": system_prompt},
            {"role": "user", "content": user_prompt},
        ],
        "temperature": temperature or self.config.temperature,
        "max_tokens": max_tokens or self.config.max_tokens,
    }
    resp = _session.post(url, headers=headers, json=payload, timeout=180)
    resp.raise_for_status()
    return resp.json()["choices"][0]["message"]["content"]
```

### 2.3 Script Agent 优化（agents/script.py）

**核心改进**：注入真实「小Lin说」脚本的 few-shot 示例

**优化内容**：
1. **开场钩子模板**：提炼小Lin说 3 种经典开场结构
   - 反常识数字 → 引发疑问 → 扭转认知
   - 新闻事件切入 → 利益相关 → 深度解析
   - 普通人视角 → 具体场景 → 宏观连接

2. **10 大叙事技巧**（含真实脚本片段示例）：
   - 抽象数据两步翻译法（「买了全球 X% 的国债」→「相当于每个中国人借了 XXX 元」）
   - 段落转折黄金句式（「但问题来了……」「这里有个细节……」）
   - 抽象→具象映射
   - 悬念前置法
   - 等等……

3. **自审循环**：Script Agent 完稿后自己打分（钩子强度/信息密度/节奏感），不达标自动重写

### 2.4 Storyboard Agent 优化（agents/storyboard.py）

**问题**：LLM 输出 JSON 格式多变，解析经常失败

**解决**：多层次 JSON 提取策略

```python
def _parse_storyboard_output(raw):
    # 1. 先找 markdown 代码块
    blocks = re.findall(r'```(?:json)?\s*\n?([\s\S]*?)\n?```', raw)
    for block in blocks:
        parsed = json.loads(block)
        # 2. 兼容嵌套格式：{"style_seeds": {...}, "storyboard": [...]}
        if "storyboard" in parsed:
            return parsed["storyboard"], parsed.get("style_seeds")
        # 3. 直接是数组
        if isinstance(parsed, list) and "segment_id" in parsed[0]:
            return parsed, None
```

**输出结构**：每个分镜段包含：
```json
{
  "segment_id": 1,
  "segment_title": "开场 — 数字钩子",
  "visual_type": "data_chart",
  "visual_description": "...",
  "script_text": "...",
  "estimated_duration_sec": 45,
  "key_elements": ["数据标注", "动态增长"],
  "text_overlay": "标题文字",
  "transition": "fade"
}
```

**架构**：

```
主线程（Streamlit）         子线程（Pipeline）
       │                          │
       │←── log_queue ────────────│ post_log()
```

**5 个审核门 UI**：

| gate_2a | 脚本大纲 Markdown | 通过 / 局部修改 / 重写大纲 |
| gate_2b | **可编辑脚本文本框** + 口播提示 | 通过（保存编辑） / 局部修改 / 重写脚本 |
| gate_3 | 分镜卡片 + 视觉风格色块 | 通过 / 局部修改 |
| gate_4 | 成片视频播放器 | 通过 / 局部修改 |

**完成页 5 个 Tab**：脚本 / 分镜表 / 视觉素材（图片网格） / 选题提案 / 知识框架

**removeChild Bug 修复**（历经 4 次迭代）：

- **根因**：`bridge.status` 在按钮提交后瞬间从 `gate` → `running` → `gate`，导致右栏在同一帧内 DOM 结构突变
- **最终方案**：`_display_mode` session_state 隔离层
  - 右栏只从 `st.session_state["_display_mode"]` 渲染（稳定）
  - 页面末尾才把 `bridge.status` 同步到 `_display_mode`
  - 过渡期（`in_transition`）保持旧显示模式，不切换
